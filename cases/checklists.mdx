## Overview

Checklists are Clerra's primary workflow management tool. They replace the deprecated case timelines system and provide flexible, customizable task management for all cases and matters.

<Note>
  **Case timelines have been deprecated.** All case tracking and workflow management now uses checklists for better flexibility and control.
</Note>

## Creating a Checklist

<Steps>
  <Step title="Open Case">
    Navigate to the case where you want to add a checklist
  </Step>

  <Step title="Go to Checklists Tab">
    Click on the **Checklists** tab in the case view
  </Step>

  <Step title="Add Checklist">
    Click **Add Checklist** button
  </Step>

  <Step title="Choose Source">
    **Option 1: From Template**
    - Select from playbook templates
    - Pre-built checklists for common case types
    - Automatically populated with standard tasks

    **Option 2: Create Custom**
    - Start with blank checklist
    - Build task list from scratch
    - Customize for unique matters
  </Step>

  <Step title="Configure Checklist">
    - **Name**: Checklist name (e.g., "Discovery Phase", "Trial Preparation")
    - **Description**: Brief overview of the checklist's purpose
    - **Phase** (optional): Associate with case phase
  </Step>

  <Step title="Add Tasks">
    Begin adding checklist items (see below)
  </Step>
</Steps>

## Checklist Items

### Creating Checklist Items

Each checklist can have multiple items with detailed properties:

**Required Fields:**
- **Task Name**: Clear description of what needs to be done
- **Status**: Not Started, In Progress, Completed, Blocked

**Optional Fields:**
- **Assignee**: Team member responsible for the task
- **Due Date**: Deadline for completion
- **Priority**: High, Medium, Low
- **Dependencies**: Tasks that must be completed first
- **Description**: Additional details or instructions
- **Attachments**: Related documents or resources

## Task Dependencies

Link tasks that must be completed in a specific order:

<Steps>
  <Step title="Select Task">
    Click on the task that depends on another
  </Step>

  <Step title="Add Dependency">
    Click **Add Dependency**
  </Step>

  <Step title="Select Prerequisite">
    Choose the task that must be completed first
  </Step>

  <Step title="Save">
    Dependent task is blocked until prerequisite is completed
  </Step>
</Steps>

**Example:**
- Task: "File motion for summary judgment"
- Dependency: "Complete legal research and draft motion"
- Result: Filing task cannot be marked complete until draft is done

## Assigning Tasks

Delegate checklist items to team members:

<Steps>
  <Step title="Select Task">
    Click on the checklist item
  </Step>

  <Step title="Assign User">
    Select team member from the dropdown (only shows case team members)
  </Step>

  <Step title="Set Deadline">
    Add due date if applicable
  </Step>

  <Step title="Notify">
    Assignee receives notification with task details
  </Step>
</Steps>

<Info>
  You can only assign tasks to team members who are already assigned to the case. Add them to the case team first if needed.
</Info>

## Due Dates & Reminders

### Setting Due Dates

<Steps>
  <Step title="Edit Task">
    Click on the checklist item
  </Step>

  <Step title="Set Due Date">
    Select date from calendar picker
  </Step>

  <Step title="Configure Reminders">
    Choose when to send reminders:
    - 1 day before
    - 3 days before
    - 1 week before
    - Custom reminder schedule
  </Step>

  <Step title="Save">
    Automatic reminders will be sent to assignee
  </Step>
</Steps>

### Deadline Tracking

System automatically:
- Sends reminders based on your settings
- Highlights overdue items in red
- Notifies case manager of overdue tasks
- Shows upcoming deadlines in dashboard

## Checklist Templates

Save time with reusable templates:

### Using Templates

<Steps>
  <Step title="Access Templates">
    When creating a checklist, select **From Template**
  </Step>

  <Step title="Choose Template">
    Browse available templates:
    - By case type (Litigation, Corporate, Real Estate)
    - By phase (Discovery, Trial, Closing)
    - Custom templates created by your firm
  </Step>

  <Step title="Customize">
    Review and modify tasks as needed for the specific case
  </Step>

  <Step title="Apply">
    Checklist is created with all template items
  </Step>
</Steps>

### Creating Custom Templates

Save your checklist as a template for future use:

1. Create and perfect a checklist
2. Click **Save as Template**
3. Name the template
4. Add description and tags
5. Set visibility (Private, Department, Workspace)
6. Template is now available for future cases

<Tip>
  Build a library of templates for your most common case types to ensure consistency and save time.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Cannot mark task as complete">
    - Check if task has incomplete dependencies
    - Verify you have permission to update the checklist
    - Ensure task is assigned to you or you're case manager
  </Accordion>

  <Accordion title="Checklist not showing progress">
    - Refresh the page
    - Verify tasks are being marked complete (not just in progress)
    - Check if sub-tasks are properly nested
  </Accordion>

  <Accordion title="Missing reminder notifications">
    - Check your notification settings
    - Verify email/in-app notification preferences
    - Ensure due dates are set correctly
  </Accordion>

  <Accordion title="Cannot assign task to team member">
    - Verify they're on the case team
    - Check if they have appropriate role/permissions
    - Add them to case team first
  </Accordion>
</AccordionGroup>
